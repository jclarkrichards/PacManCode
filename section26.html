Section 26: Displaying Text

Setup

We need to be able to add text to our game so that we can display the scores, level, and any other information we want to convey to the player. Like everything else in the game, the text is just sprite images in the spritesheet. As you can see the only text we will use are the numbers 0-9 and the upper case letters A-Z. We will not use lower-case letters or puncuation. If you want puncuation, then go to school! If you're game includes a story which requires you to write sentences, then you'll want some puncuation images as well. But we are not going to include a story in our game, just a few words at a time. In any case we need to be able to input a word or two and output the correct images to represent that word in the correct location.

The nice thing about text is that there's a one-to-one relationship between the text and the images. There's only one possible image for an uppercase A and only one image for the number 5 for example. So we can just put the images into a dictionary. When we receive a word a some string to convert into the images, we just look them up using the dictionary. Making the dictionary is a pain though because we'll just have to specify exactly where to find each image on the spritesheet for each character. It's tedious, but I don't know any other way to automate this process. I mean the letters and numbers can be anywhere in the spritesheet and in any order. So sometimes you just have to do some boring coding like this.
